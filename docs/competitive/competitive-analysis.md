## LiteTerm 的差异化优势

1. **开源 + 完全免费 + 三平台** — Warp 不支持 Windows,FinalShell/XShell/SecureCRT 闭源
2. **系统监控 + 进程管理器** — 只有 FinalShell 做到这个级别,但 FinalShell 闭源且 Java 吃内存
3. **ZMODEM + SFTP 双通道** — 堡垒机场景独有(SFTP 到不了目标机时回退 rz)
4. **终端录屏/回放** — 多数竞品不支持
5. **AES-256-GCM 密码存储** — 比 FinalShell(DES)和 XShell(固定密钥)更安全

## 竞品有但 LiteTerm 未实现的功能

### 优先级高(用户价值大,实现可行)
| 功能 | 竞品参考 | 预估工作量 |
|------|----------|-----------|
| AI 命令生成/错误解释 | Warp | 已设计,待对接推理后端 |
| 非活跃标签 scrollback 缩减 | — | 1 天 |
| 快捷键自定义(完善) | Tabby/iTerm2 | 已有基础,需扩展 |

### 优先级中(有价值但不急)
| 功能 | 竞品参考 | 预估工作量 |
|------|----------|-----------|
| RDP/VNC 远程桌面 | MobaXterm | 较大(需新协议栈) |
| X11 转发 | MobaXterm/SecureCRT | 中等 |
| 插件系统 | Tabby | 较大(架构改造) |
| 多会话同步输入 | SecureCRT/XShell | 1-2 天 |

### 不做(投入产出比低)
| 功能 | 原因 |
|------|------|
| 移动端 | 无竞品做到,市场需求不明确 |
| Telnet/TFTP | 过时协议,几乎没人用 |
| 内置文本编辑器 | 用户有 vim/nano,不需要终端客户端内置 |
| Windows WSL 集成 | Warp 做了但反馈一般,Windows Terminal 更合适 |
